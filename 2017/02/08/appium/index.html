基于Appium的输入法自动化测试

第三方支持库

  • Robot Framework (http://www.robotframework.org)
  • Appium (http://www.appium.io/)
  • AppiumLibrary (https://github.com/jollychang/robotframework-appiumlibrary)
  • WxPython (http://sourceforge.net/projects/wxpython/files/wxPython/2.8.12.1/)
  • RIDE (https://github.com/robotframework/RIDE/wiki)
  • Selenium2Library (https://github.com/robotframework/Selenium2Library)
  • Pywin32 (https://sourceforge.net/projects/pywin32/?source=typ_redirect)
  • AutoIt (https://www.autoitscript.com/site/autoit)
  • AutoItLibrary (https://code.google.com/p/robotframework-autoitlibrary/)

环境搭建

  • 安装最新的python2.7版本（32位和64位根据自身操作系统选择）
    • 如果是linux环境推荐使用vituralenv等工具维护不同的python版本
    • python安装路径不能有空格和其他特殊字符
    • 安装完成后请检查环境变量。PATH路径中加入python安装路径和$PYTHONPATH\Scripts

  • 最新版本的python已经集成了pip工具，如果使用的旧版本，请先安装setuptools和pip工具

    • 安装python扩展包安装工具setuptools

    • https://pypi.python.org/pypi/setuptools/

    • 安装python扩展包管理工具pip

    • https://bootstrap.pypa.io/get-pip.py
    • python get-pip.py
  • 下载安装WxPython，Pywin32，AutoIt。请选择对应的32位或64位的版本，必须和python版本一致
  • 通过pip安装其他扩展包
    • pip install robotframework-AppiumLibrary
    • pip install robotframework-Selenium2Library
    • pip install robotframework-ride
    • pip install AutoItLibrary
    • 或者升级已经安装的版本，使用 pip install –upgrade 命令
    • 以下依赖包会在安装过程中自动安装
      • decorator
      • docutils
      • Appium-Python-Client
      • selenium
      • mock
      • sauceclient
      • pytest-cov
      • pytest-xdist
      • pytest-pythonpath

  • 安装Appium

    • 使用Node.js安装

    • 使用GUI版本

      • 下载安装版本
    • 使用源码安装
      • 下载安装Node.js

如何运行

  • 将输入法测试库加入python的lib path中
    • 在\Python27\Lib\site-packages目录新建一个Ifly.pth文件
    • 编辑内容为存放自动化代码的路径(iFly目录的上级目录)
  • 在被测手机上安装带有按键日志的输入法版本
    • 请确认最新的代码是否有按键日志，如果没有请安装TestApp目录下app-debug-log.apk
  • 使用命令行运行
    • 进入测试用例目录，执行测试用例
    • 执行指定目录的测试用例
      • pybot BVT
    • 执行指定的测试集合
      • pybot 九键拼音.txt
    • 执行指定的测试用例
      • pybot -t 九键拼音输入汉字 九键拼音.txt
    • 在指定的设备上执行用例
      • pybot -v udid:000007CF008B869A -t 九键拼音输入汉字 –loglevel DEBUG 九键拼音.txt
    • 指定运行还有某个TAG的用例
      • pybot -i BVT InputTest
    • 执行上次运行时失败的用例
      • pybot -R output.xml BVT
  • 使用RIDE工具执行用例
    • 在RIDE工具中打开用例目录，加载完成后可以选择用例执行
    • 在Run Tab页中的Arguments输入框中可以指定相关运行参数

代码结构

  • iFly目录用于存放python关键字，各类型关键字按文件区分。

  • 关键字名称

    1. 各个文件中的关键字应尽量保证全局唯一。由于Robot在运行过程中不会主动区分class，不同class下使用相同方法名会导致robot无法正确识别
    2. 关键字使用下划线分割小写字母的方式命名。robot运行过程中会将空格转成下划线进行关键字匹配
    3. 关键字必须提供使用说明，需要提供参数说明，返回值，使用实例等信息。
    • 注释实例：
    1. 不开放的函数应该以下划线开头。例如： _hidden_function
    2. 开放的关键字统一在iFly目录下init.py文件中定义
  • 更新关键字文档
    * 添加或修改关键字后可以使用以下命令更新文档
* python -m robot.libdoc CustomLibrary.NewsCustomLibrary NewsCustomLibrary.html
  • 更新pyc文件
    * python -m py_compile AppiumLib.py

用例结构

  • InputTest目录用于存放robot测试用例和资源文件

    • BVT – 存放BVT测试用例
    • Keywords – 存放Robot关键字
    • Resources – 存放环境变量和页面元素相关信息

  • 测试用例

    1. 用例安装功能模块进行划分，每个特性使用单独的目录存放，特殊的独立测试也可以新建目录存放
    2. 每个文件是一个测试集合，测试集合根据用例相关性进行组合。通常是一组有相同预置条件的用例
    3. 测试用例尽量相互独立，避免相互依赖。如果存在特殊情况可使用${PREV TEST STATUS}变量检查前一个用例的执行状态
    4. 用例一般包含准备，执行，验证，清理四个阶段，一个用例必须要有验证阶段。
    5. 用例名称及用例集使用中文描述
    6. 用例名尽量简短，简要描述测试目标
    7. 用例中的操作步骤关键字应该尽量通俗易懂，屏蔽次要的信息，尽量避免复杂的逻辑。
    8. 一个用例尽量只测试一个场景
    9. 用例有三种组织形式，根据需要选择。新特性可以考虑采用BDD用例模式。
      • 流程驱动用例
      • 数据驱动用例
      • BDD用例
    10. 尽量避免采用Sleep关键字同步测试运行状态，可以选择的情况下尽量采用Wait类关键字检查执行状态。

  • 用例TAG

    1. TAG可以在目录上面添加，添加后该目录下说有用例都会继承这个TAG
    2. TAG也可以在文件（用例集）上添加。两种添加方式 Force Tags 和 Default Tags
      • Force Tags – 用例集下所有用例均继承这个TAG
      • Default Tags – 用例集下没有TAG的用例会使用这个TAG
    3. 每个用例都需要有TAG，TAG需要区分以下信息：
      • 测试特性 – 如 拼音，笔画，手写等
      • 测试类型 – BVT, 功能， 性能，稳定性
      • 测试优先级 – level1 ~ level5
      • 特殊测试标识 – 如 deprecated，notFinished
      • 特殊环境标识 – 如 SD卡，WIFI，3G，服务器交互，Root等

ROBOT关键字

关键字按功能模块分文件存放。Robot关键字分为基础关键字和业务关键字。两种关键字可放在同一个文件中，按次序存放，先写基础关键字，后写业务关键字。关键字需要保证全局唯一，不能重复。

  • 基础关键字
    • 由于基础关键字很容易重复，因此采用特性名（或页面）+关键字的方式定义。例如：
      • New Message Page Set Title
      • New User Group Page Set Group Name
    • 基础关键字用于封装单个页面元素的操作，这样有利于后期的维护，当某个元素的定位信息发生变化，而整个操作流程不变的情况下，只需要修改这个基础关键字
    • 原则上页面的每个元素都要封装基础操作的关键字

  • 业务关键字

    • 由多个基础关键字组成的操作序列，用于实现某个特定的业务功能。例如：
      • User Group Page Search User Group
      • Tab Navigate to Page
    • 业务关键字尽量添加前缀，在保证不引起混淆的情况下可以不加特性或页面的前缀。
    • 业务关键字操作完成后尽量返回起始页面，如果没有，必须在注释中说明
    • 重要的业务关键字必须要有注释，包含以下信息：
      • 功能说明
      • 参数说明
      • 返回值说明
      • 起始页面
      • 结束页面
      • Examples

    • 例如：

    • 可以通过以下命令生成和更新业务关键字文档：

      • python -m robot.libdoc Keywords/Navigate.txt doc/Navigate.html
    • 复杂的关键字可以考虑使用python实现，放在lib库中
    • 业务关键字按照页面或特性划分，按文件存放

Resource管理

Resources目录主要包含环境变量信息和元素定位信息

  • 变量
    • 变量可以使用 -v 参数在运行时进行重载。例如可以在运行时指定设备，服务器地址，登录账号等信息。从而避免频繁的配置更新操作
    • 全局变量采用大写字母，局部变量采用小写字母

Appium常用运行参数

* appium.js -p port -U deviceId --session-override --log-level error
